# Health Check Lab Performance Report

## 目的

スマホ4G回線でも初回表示を軽くするため、デザイン変更より速度を優先して調整しました。

## 実施内容

- 初回表示時のSupabase集計取得を待機時間に回しました。
- 結果画面での集計再取得も、画面描画後の待機時間に回しました。
- smooth scrollを削除し、即時スクロールに変更しました。
- 背景や選択状態の装飾グラデーションを単色化しました。
- headerのぼかし効果を削除しました。
- 影の描画を実質無効化しました。
- 画面外セクションに `content-visibility: auto` を追加しました。
- JS/CSSのキャッシュヘッダーをNetlify向けに追加しました。
- JS読み込みに `defer` を追加しました。

## 計測

| 項目 | 変更前 | 変更後 |
| --- | ---: | ---: |
| app.js | 55,215 bytes | 55,437 bytes |
| styles.css | 16,229 bytes | 15,361 bytes |
| index.html | 2,655 bytes | 2,653 bytes |
| ローカル `/body-check` 応答 | 104.55 ms | 63.56 ms |

`app.js` は待機実行用の小さな関数を追加したため微増していますが、初回表示時の通信と描画負荷を下げています。

## 残したもの

- 結果の円形スコア表示は、結果の分かりやすさに必要なため残しました。
- OpenAI API連携用の既存準備コードは削除していません。
- Supabase連携は機能として残しつつ、初回表示の邪魔をしにくい形にしました。

## 次の高速化候補

- `app.js` をページ別ファイルに分割して、初回はトップページ分だけ読む。
- CSSをページ別に分割する。
- 公開後にChrome Lighthouseで実機に近い速度計測を行う。
